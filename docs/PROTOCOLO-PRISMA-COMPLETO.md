## 2. ANTECEDENTES E OBJETIVOS

### 2.1 Rationale

**Por que esta revisão é importante?**

As micro e pequenas empresas (MPEs) são críticas para a economia brasileira, representando ~99% das empresas formais. No entanto, enfrentam desafios em transferência de conhecimento e inovação. Enquanto há extensa literatura sobre absorptive capacity e competitividade, há LACUNAS específicas:

1. **Contexto Brasil**: Poucos estudos em contexto brasileiro específico
2. **MPEs**: Maioria dos estudos foca grandes empresas
3. **Mecanismos**: Como redes de inovação podem compensar baixa AC em MPEs
4. **Dinâmica temporal**: Faltam estudos longitudinais sobre evolução de AC

**Impacto esperado**: Identificar lacunas, sugerir problemas de pesquisa, fundamentar mestrado

### 2.2 Objetivos

#### Objetivo Primário
Mapear sistematicamente o estado do conhecimento sobre transferência de conhecimento externo, absorptive capacity e competitividade em MPEs brasileiras, através de revisão de literatura entre 2015-2026.

#### Objetivos Secundários

1. Identificar teorias principais que explicam relação entre transferência de conhecimento e competitividade
2. Mapear como conceitos-chave (AC, DC, OL) são medidos e operacionalizados
3. Identificar fatores institucionais, contextuais que facilitam/dificultam transferência em Brasil
4. Detectar lacunas específicas na literatura
5. Formular problemas de pesquisa emergentes para mestrado

### 🤖 5.4 Protocolo de Validação com IA e Krippendorff's Alpha

#### Justificativa
Não é possível ter 2 revisores humanos lendo todos os 180 artigos em full-text (custo-proibitivo, tempo excessivo).

**SOLUÇÃO**: Usar 2 modelos de IA DIFERENTES para validar mutuamente:
- **Claude 3.5 Sonnet** (Modelo 1)
- **Gemini 2.0 Pro** (Modelo 2)

#### Protocolo Detalhado

**PASSO 1: Processamento Paralelo**
```
Para cada artigo no Nível 2 (Full-Text):
  
  1a. Claude processa:
      - Lê full-text PDF
      - Extrai dados no formato JSON
      - Gera fichamento estruturado
      - Output: fichamento_claude.json
  
  1b. Gemini processa (INDEPENDENTEMENTE):
      - Lê mesmo full-text PDF
      - Extrai dados no formato JSON
      - Gera fichamento estruturado
      - Output: fichamento_gemini.json
  
  Nota: IAs processam em paralelo, sem ver output uma da outra
```

**PASSO 2: Cálculo de Concordância (Krippendorff's Alpha)**

Método baseado em **Krippendorff's Alpha (α)** (concordância inter-rater robusta):

```
Krippendorff's Alpha = 1 - (Do / De)

Onde:
- Do = discordância observada (erro real)
- De = discordância esperada por acaso

Recomendado por PRISMA 2024-IA para validação multi-avaliador com IA
Referência: Krippendorff, K. (2004). Content Analysis: An Introduction to Its Methodology
```

**Campos comparados** (12 campos principais):
1. Objetivo do estudo (tema principal)
2. Design estudo (qualitativo/quantitativo/misto)
3. Amostra (N participantes)
4. Contexto geográfico
5. Setor industrial
6. Fonte conhecimento externo
7. Medida AC (dimensão: PAC/RAC)
8. Indicador competitividade
9. Achado principal (relação confirmada S/N)
10. Limitações metodológicas
11. Brasil mencionado (S/N)
12. Classificação qualidade (Baixa/Moderada/Alta)

**Interpretação** (Krippendorff's Alpha):
| Alpha | Concordância | Ação |
|--------|-------------|------|
| 0.80 - 1.00 | **Excelente** ✅ | USE (aprovação automática) |
| 0.70 - 0.79 | **Boa** ✅ | USE com revisão 10% |
| 0.60 - 0.69 | **Aceitável** 🔍 | REVISE amostra 30% |
| < 0.60 | **Fraca** ❌ | DESCARTE (reprocesse) |

**PASSO 3: Ações Baseadas em Alpha**

```
Se Alpha ≥ 0.80 (EXCELENTE):
  → Confie na IA
  → Use fichamento de qualquer modelo
  → Não precisa revisão humana
  → Time savings: 15 min por artigo

Se 0.70 ≤ Alpha < 0.80 (BOA):
  → Revisar amostra 10%
  → Pesquisador valida pontos críticos
  → Time: 10 min por artigo revisado

Se 0.60 ≤ Alpha < 0.70 (ACEITÁVEL):
  → Marcado para revisão humana
  → Pesquisador resolve discordâncias
  → Toma decisão final
  → Time: 20 min por artigo

Se Alpha < 0.60 (FRACA):
  → Artigo reprocessado com prompts refinados
  → Ambas IAs reexecutam
  → Se alpha ainda < 0.60: considere exclusão
```

**PASSO 4: Amostra Piloto de Validação**

Antes de processar 180 artigos:
- Processe 10 artigos pilotos com ambas IAs
- Calcule Krippendorff's Alpha média
- Se média ≥ 0.70: proceda com confiança
- Se média < 0.70: refine prompts e repita piloto

### 🤖 6.4 Design de Prompts e Engenharia de IA

#### Filosofia de Design

Os prompts foram projetados seguindo princípios de **Prompt Engineering**:

1. **Especificidade** - Instruções exatas, não genéricas
2. **Estrutura** - Formato JSON para análise automática
3. **Few-Shot Examples** - Exemplos de output esperado
4. **Guardrails** - Limites claros (não saia do artigo)
5. **Versioning** - Rastreabilidade de mudanças

#### Componentes de Cada Prompt

**HEADER** (Contexto):
```
Você é um pesquisador experiente em gestão organizacional 
analisando artigos sobre transferência de conhecimento, 
capacidade absortiva e competitividade em pequenas empresas.
```

**INSTRUÇÕES** (O que fazer):
```
1. Leia o artigo completo fornecido
2. Extraia informações nos 12 campos obrigatórios
3. Se não encontrar informação: use "Não mencionado"
4. Não adicione interpretações fora do texto
5. Retorne resultado em JSON válido
```

**GUARDRAILS** (Limites):
```
IMPORTANTE:
- NÃO interprete o que não está escrito
- NÃO use conhecimento externo
- NÃO extraia dados de referências
- USE APENAS informações no artigo fornecido
```

#### Versioning e Rastreabilidade

Todos os prompts salvos em:
- **`scripts/utils/prompts.py`** - Código Python com prompts
- **`scripts/utils/prompts_version_log.md`** - Histórico de alterações

Cada versão tem:
- Versão (v1.0, v1.1, etc)
- Data
- Alterações
- Motivo da mudança
- Pesquisador que mudou

### 🤖 7.4 Viés Específico de IA

#### Vieses Conhecidos de Grandes Modelos de Linguagem

| Tipo de Viés | Descrição | Mitigação |
|-------------|-----------|-----------|
| **Alucinação** | IA inventa dados não existentes | Validação Krippendorff's Alpha (detecta inconsistências) |
| **Viés de treinamento** | Dados até Mar/2026 | Limitar a contexto 2015-2026 (dentro janela) |
| **Viés linguístico** | Favorece idioma de treinamento | Usar prompts PT-BR + EN estruturados |
| **Viés de recência** | Favorece papers recentes | Analisar distribuição temporal |
| **Viés de popularidade** | Favorece autores conhecidos | Cegação por nome do autor (se possível) |

#### Protocolo de Detecção de Viés IA

**TESTE 1: Alucinação**
- 5% amostra dos fichamentos comparada com leitura manual
- Verificar se campos têm dados "inventados"
- Se taxa alucinação > 2%: refinar prompt

**TESTE 2: Consistência**
- Reprocesse 10 artigos com mesma IA após 1 semana
- Compare Krippendorff's Alpha entre runs
- Deve ser > 0.95 (alta reprodutibilidade)

**TESTE 3: Viés de Seleção**
- Analyze Krippendorff's Alpha por setor industrial
- Se alpha varia muito entre setores: investigar viés
- Ajustar prompts se necessário

## 8. MÉTODO DE SÍNTESE

### 8.1 Síntese Qualitativa

Dado o objetivo de identificar LACUNAS e PROBLEMAS DE PESQUISA (não teste de hipóteses), será utilizada:

**Análise Temática Estruturada**:
1. Ler todos os fichamentos
2. Extrair temas/padrões principais
3. Mapear conceitos-chave e relacionamentos
4. Identificar divergências
5. Documentar lacunas
6. Formular problemas pesquisa emergentes

**Ferramentas**:
- Matriz temática (N artigos × temas)
- Mapa conceitual (relações entre conceitos)
- Síntese narrativa (integração qualitativa)

### 8.2 Síntese Quantitativa (Descritiva)

- Distribuição temporal (artigos por ano)
- Distribuição geográfica
- Distribuição por periódico
- Distribuição por tipo estudo (quali/quanti/misto)
- Frequência de temas

### 8.3 Meta-análise

**NÃO será feita meta-análise** pois:
- Heterogeneidade metodológica alta
- Objetivo é síntese qualitativa (identificar lacunas)
- Diversidade de medidas de AC e competitividade
